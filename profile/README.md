## Resolving scan and setup friction

Authentication gaps: verify login steps, session handling, test credentials, and role permissions before running a Probely web vulnerability scanner job.  
Crawler misses: add known routes, seed URLs, or API definitions so Probely web security checks can reach deeper application areas.  
Too many duplicate findings: adjust target structure and triage rules in Probely vulnerability management to keep reports focused.  
API coverage issues: confirm endpoint documentation, authentication headers, and environment variables for Probely API security scans.
